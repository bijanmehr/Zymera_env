Tutorial 2 · A network to control the agent

We put a neural network in charge of the agents and train it. The whole thing is one runnable file — examples/learn_template.py — and this page walks through it piece by piece. The shape of every interface (obs, the step tuple, the trajectory) is from Tutorial 1. In MARL-textbook terms: a policy network πi trained by policy gradient (Ch 2, 8).

The pipeline is a loop of five parts. You write the three in the middle; the env and rollout are given:

   env ──reset──► obs ──► [ NETWORK ] ──► action ──► step ──► trajectory
                                                                  │
                          ┌──────────────────────────────────────┘
                          ▼
                  [ REWARD ] ──► [ LOSS ] ──► grad ──► optax update ──► better network
                                                                              │
                          ◄───────────────── repeat ──────────────────────────┘

The env (use the pure-JAX one)

Training drives the env through rollout, which traces it inside lax.scan. So use the pure-JAX env — keep sense_radius=0 (the default). The occlusion Sensor is host-side and can't be traced; it's for the Python-loop agents, not for training.

import jax, jax.numpy as jnp, equinox as eqx, optax, zymera
env = zymera.make("empty-v0", grid_h=12, grid_w=12, n_agents=2)   # sense_radius=0

The network — a policy is a callable obs → action

The env only asks for a callable policy(obs, key) → (N,) int32. A network is just a fancier one: run an MLP per agent to get logits, sample an action.

class Policy(eqx.Module):
    net: eqx.nn.MLP
    def __init__(self, obs_dim, n_actions, key):
        self.net = eqx.nn.MLP(obs_dim, n_actions, width_size=64, depth=2, key=key)
    def __call__(self, obs, key):                 # obs (N, obs_dim) -> (N,) int32
        logits = jax.vmap(self.net)(obs)          # (N, n_actions)   per-agent logits
        return jax.random.categorical(key, logits, axis=-1).astype(jnp.int32)

policy = Policy(env.obs_dim, env.n_actions, jax.random.PRNGKey(0))

One shared MLP runs on each agent's obs row independently (jax.vmap). The same net gives the logits the loss needs later.

Collect experience — rollout

rollout runs the policy for n_steps and stacks everything into a trajectory pytree:

traj = zymera.rollout(env, policy, n_steps=64, key=key)

traj["world"]    # World pytree, leaves (T+1, ...)   — every state along the way
traj["obs"]      # (T+1, N, obs_dim)                 — what the policy saw
traj["action"]   # (T,   N)      int32               — what it did
traj["reward"]   # (T,   N)      float32             — ZEROS (env has no reward)
traj["done"]     # (T,   N)      bool

Note the alignment: action[t] was chosen from obs[t], so traj["obs"][:-1] pairs with traj["action"] — the loss uses exactly that.

The reward — your objective, from the trajectory

The env's reward is zeros: you define the objective, as a function of the trajectory. Here: drive the agents to the bottom-right corner — reward each agent for getting closer each step.

GRID, GOAL = 12, (11, 11)

def reward_fn(traj):                              # -> (T, N) float32
    pos  = traj["world"].body.position.astype(jnp.float32)        # (T+1, N, 2)
    dist = jnp.abs(pos - jnp.array(GOAL, jnp.float32)).sum(-1)    # (T+1, N) Manhattan
    return dist[:-1] - dist[1:]                                   # closer = positive

What's learnable depends on the obs. [row, col, coverage] is enough to learn "go to the corner", but NOT to learn good coverage — the policy can't see where it hasn't been. For that, give it a richer obs (override sense in a custom env, see Tutorial 1).

The loss — REINFORCE

Pull up the log-probability of actions that led to high return. Recompute logits on the stored obs, gather the taken actions, weight by the (normalised) discounted return:

def discounted_returns(rewards, gamma=0.99):      # (T, N) -> (T, N)
    def step(carry, r): carry = r + gamma * carry; return carry, carry
    _, g = jax.lax.scan(step, jnp.zeros_like(rewards[0]), rewards[::-1])
    return g[::-1]

def loss_fn(policy, traj):
    obs, act = traj["obs"][:-1], traj["action"]   # (T, N, od), (T, N)
    T, N, od = obs.shape
    logits = jax.vmap(policy.net)(obs.reshape(T * N, od)).reshape(T, N, -1)
    logp   = jax.nn.log_softmax(logits, axis=-1)
    chosen = jnp.take_along_axis(logp, act[..., None], axis=-1)[..., 0]   # (T, N)

    G = discounted_returns(reward_fn(traj))
    G = (G - G.mean()) / (G.std() + 1e-8)         # variance-reducing baseline
    return -jnp.mean(chosen * G)                  # REINFORCE objective

Putting it together — the training step + loop

One jitted step: roll out, differentiate the loss w.r.t. the network, apply an optax update. Then loop.

opt = optax.adam(3e-3)
opt_state = opt.init(eqx.filter(policy, eqx.is_array))

@eqx.filter_jit
def train_step(policy, opt_state, key):
    traj = zymera.rollout(env, policy, 64, key)               # collect
    loss, grads = eqx.filter_value_and_grad(loss_fn)(policy, traj)
    updates, opt_state = opt.update(grads, opt_state)         # optimise
    policy = eqx.apply_updates(policy, updates)
    return policy, opt_state, loss

for it in range(200):
    key, sk = jax.random.split(key)
    policy, opt_state, loss = train_step(policy, opt_state, sk)

Run examples/learn_template.py and the agents learn to reach the corner — average distance falls to zero:

Swapping machinery

To try something different, edit one box — everything else stays:

keys = jax.random.split(key, 8)

batch["action"].shape       # (8, 64, n_agents)
