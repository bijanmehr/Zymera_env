Tutorial 1 · The environment

Make a world, change its settings, add agents, and read what an agent perceives. Every snippet runs as-is. When you're ready to put a learned network in control, go to Tutorial 2.

In the language of the MARL textbook (Ch 3) this is a partially observable multi-agent environment — a POSG, or a Dec-POMDP when the agents share one reward; each agent receives its own observation oi.

Create a world & change its settings

One factory call gives you a gym-style env. reset returns (obs, state).

import jax, zymera
key = jax.random.PRNGKey(0)

env = zymera.make("empty-v0", grid_h=8, grid_w=8, n_agents=1)
obs, state = env.reset(key)

The world is a grid of cells. Agents and walls sit on it, each addressed by (row, col) — row grows downward:

            col →   0    1    2    3
    row 0           ·    ·    #    ·
     │   1          ·    A    ·    ·
     ▼   2          ·    ·    ·    ·

    A = agent at (row=1, col=1)     # = wall     · = free cell

Every setting is a keyword to make:

env = zymera.make("empty-v0",
                  grid_h=16, grid_w=16,   # world size
                  n_agents=4,             # number of agents
                  n_obstacles=20,         # random walls (or wall=)
                  sense_radius=3,         # per-agent view; 0 = no sensor (default)
                  occlusion=True)         # walls block line of sight
settingeffect
grid_h, grid_wgrid dimensions
n_agentshow many agents (drives every shape — see below)
n_obstacles / wallrandom count, or an explicit (H,W) bool mask
sense_radius / occlusionattach a partial-obs Sensor (see What the agent perceives)

Agents & config

n_agents sets every per-agent shape; nothing else to wire.

import jax.numpy as jnp
env = zymera.make("empty-v0", grid_h=12, grid_w=12, n_agents=8)
env.n_agents, env.obs_dim, env.n_actions      # → (8, 3, 5)

obs, state = env.reset(key)
obs.shape                                      # (8, 3)  — one obs row per agent
state.body.position.shape                      # (8, 2)  — (row, col) per agent

# one ActionId per agent: STAY=0 NORTH=1 EAST=2 SOUTH=3 WEST=4
action = jnp.full((env.n_agents,), int(zymera.ActionId.EAST), jnp.int32)
obs, state, reward, done, info = env.step(state, action, key)

The step return is a 5-tuple — (obs, state, reward, done, info), with reward and done shaped (N,). (reward is zeros — you define the objective in Tutorial 2.)

For a controlled config, pass an explicit obstacle mask instead of a random count:

import numpy as np
wall = np.zeros((12, 12), bool); wall[4:9, 6] = True     # a vertical bar
env = zymera.make("empty-v0", grid_h=12, grid_w=12, n_agents=2, wall=wall)

The state pytree

state is one JAX pytree — it threads through jit / vmap / lax.scan unchanged. Its fields:

fieldshapewhat
state.body.position(N,2) int32per-agent (row, col)
state.explored(H,W) int32per-cell visit count
state.step_count() int32ticks since reset
state.wall(H,W) boolobstacles (movement enforces them)
state.seen_by(N,H,W) boolper-agent visibility — filled only in the host-side Sensor path (sense_radius ≥ 1); never during training/rollout
state.comm_graph(N,N) booladjacency (placeholder for now)

What the agent perceives

"Observation" means two different things in zymera — read both off one concrete world:

The world. A 5×5 grid. Our agent A stands at (row 2, col 1). Two cells are walls. Four cells have been visited so far (the three marked ° plus the cell A is standing on):

           col  0    1    2    3    4
    row 0   ·    ·    ·    ·    ·
    row 1   °    ·    #    ·    ·
    row 2   °    A    #    ·    ·
    row 3   °    ·    ·    ·    ·
    row 4   ·    ·    ·    ·    ·

    A = agent      # = wall      ° = already visited      · = free, unvisited

Observation #1 — obs (what a learning policy sees)

Each step, reset / step hand your policy a tiny fixed vector — 3 numbers per agent. For our agent you read them straight off the picture:

obs  =  [ 2.0 ,  1.0 ,  0.16 ]
          │      │       │
          row    col     coverage  =  4 of 25 cells visited  =  0.16
          └─ where the agent is ─┘   (one global number, the SAME for every agent)

That is the entire learning input for "empty-v0": where am I + how much of the map is done. Look at what is not there — the policy does not see the walls, its neighbours, or where other agents are. It's a deliberately thin signal you train a network against.

Observation #2 — the Sensor reading (what a hand-written agent sees)

An agent in a Python loop can instead ask the Sensor what's around it. With sense_radius=2 it gets the occupancy of every cell within 2 steps. Marking each sensed cell s on the same world:

           col  0    1    2    3    4
    row 0   ·    s    ·    ·    ·
    row 1   s    s    #    ·    ·       s = a cell the Sensor reports back
    row 2   s    A    #    s    ·       # = wall  → reported BLOCKED
    row 3   s    s    s    ·    ·       A = agent (it also senses one cell off
    row 4   ·    s    ·    ·    ·            the grid to its far west → an edge)

The reading is just a dictionary, {cell: blocked}, where blocked = True means "wall or off-grid — can't step there" and False means "free — I could step here":

env.sensor.observe_one(state, 0)  →  {
    (0,1): False,  (1,0): False,  (1,1): False,    # free  → I could go here
    (2,0): False,  (2,3): False,  (3,0): False,
    (3,1): False,  (3,2): False,  (4,1): False,
    (1,2): True,   (2,2): True,                     # the two walls  → blocked
    (2,-1): True,                                   # off the grid   → blocked
}

This is what a reactive agent runs on (see examples/lawn_mower.py). It's purely local: the agent never gets the grid size or the wall map — only "what's blocked right around me", rebuilt every step.

Sense range — sense_radius

The example above used sense_radius=2. The radius is simply how big that disk of sensed cells is — a Euclidean disk around the agent. Bigger radius → more cells reported each step → less guessing:

  radius 1 (4 cells)     radius 2 (12 cells)        radius 3 (28 cells)

      ·  X  ·             ·  ·  X  ·  ·              ·  ·  ·  X  ·  ·  ·
      X  A  X             ·  X  X  X  ·              ·  X  X  X  X  X  ·
      ·  X  ·             X  X  A  X  X              ·  X  X  X  X  X  ·
                          ·  X  X  X  ·              X  X  X  A  X  X  X
                          ·  ·  X  ·  ·              ·  X  X  X  X  X  ·
                                                     ·  X  X  X  X  X  ·
                                                     ·  ·  ·  X  ·  ·  ·

   A = agent     X = a cell the Sensor reports     blocked = wall OR off-grid

Occlusion — do walls block the view?

occlusion=True makes walls cast shadows: a cell is sensed only if the straight line to it is clear (you see the wall's face, not what is behind it). occlusion=False is an x-ray disk.

   occlusion OFF (x-ray)            occlusion ON (line of sight)

     ·  ·  ·  ·  ·  ·  ·               ·  ·  ·  ·  ·  ·  ·
     ·  ·  ·  ·  ·  ·  ·               ·  ·  ·  ·  ·  ?  ?
     ·  ·  ·  A  #  ·  ·               ·  ·  ·  A  #  ?  ?     # blocks the view;
     ·  ·  ·  ·  ·  ·  ·               ·  ·  ·  ·  ·  ?  ?     cells behind it (?)
     ·  ·  ·  ·  ·  ·  ·               ·  ·  ·  ·  ·  ·  ·     are unknown

   A = agent     # = wall     ? = shadowed (reported as unknown / absent)

The Sensor is a host-side tool for Python-loop agents, not for training — Tutorial 2 explains why training uses the pure-JAX env.

A custom world

Need different dynamics or a richer obs? Subclass zymera.Env and register it:

class MyEnv(zymera.Env):
    def __init__(self, grid_h=12, grid_w=12, n_agents=3):
        super().__init__(n_agents=n_agents)
        self.grid_h, self.grid_w = grid_h, grid_w
    def reset(self, key):
        state = zymera.World.initial(key, self.grid_h, self.grid_w, self.n_agents)
        return state.sense(), state         # customise obs by overriding sense
    # step is inherited

zymera.register_env("my-env-v0", MyEnv)
env = zymera.make("my-env-v0", n_agents=2)
